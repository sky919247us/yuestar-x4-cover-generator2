閱星曈X4 待機畫面網頁版生成器 - 軟體規格書 (SRS)
版本： 2.0 日期： 2025年10月20日

1. 簡介
1.1 目的 (Purpose)
本文件旨在清楚、完整地定義「閱星曈X4 待機畫面網頁版生成器」的功能性與非功能性需求。此線上工具讓使用者能輕易上傳、編輯、添加文字並合成個人化圖文內容，最終輸出一張符合閱星曈X4電子書閱讀器待機畫面規格 (480 x 800 像素) 的 JPG 圖片。

1.2 專案範疇 (Scope)
本專案為一個前端驅動的網頁應用程式 (Web Application)。所有圖像與文字處理操作皆在使用者端的瀏覽器中完成。其核心功能包含：

範本選用與素材上傳。
圖像與文字的編輯：旋轉、縮放、移動。
針對 E-ink 螢幕的專業圖像處理：灰階轉換、抖動處理。
圖層管理與內容合成。
符合裝置規格的圖片輸出與下載。
1.3 目標受眾 (Target Audience)
本工具的主要使用者為擁有「閱星曈X4」電子書閱讀器的用戶，他們希望能夠不需專業繪圖軟體，即可快速製作具備高度個人化的待機畫面。

2. 整體描述
2.1 產品願景 (Product Perspective)
這是一個獨立、免費的線上工具，旨在提升閱星曈X4用戶的個人化體驗。它應具備直覺、易用的操作介面，並提供範本以降低創作門檻，同時也具備圖層管理、文字編輯等進階功能，滿足不同程度使用者的需求。

2.2 使用者特徵 (User Characteristics)
熟悉基本的網頁瀏覽與檔案上傳操作。
希望快速、方便地客製化自己的設備。
可能在桌上型電腦或行動裝置（手機、平板）上使用此工具。
2.3 設計與實現約束 (Design and Implementation Constraints)
技術棧： 建議使用 HTML5, CSS3, JavaScript。可考慮使用如 Canvas API 進行圖像處理，或引入成熟的 JavaScript 圖片處理函式庫（如 Fabric.js, Konva.js）以加速開發。
客戶端處理： 為降低伺服器負擔與成本，所有圖片處理邏輯皆在使用者瀏覽器中執行。
跨瀏覽器兼容性： 應支援所有主流現代瀏覽器（Chrome, Firefox, Safari, Edge）的最新版本。
響應式設計 (RWD)： 介面必須能自動適應不同尺寸的螢幕，確保在手機和電腦上都有一致且良好的使用體驗。
3. 詳細需求 (Specific Requirements)
3.1 功能需求 (Functional Requirements)
3.1.1 內容輸入與範本 (Content Input & Templates)

FR-1.1：主要圖片上傳

系統應提供一個明確的按鈕或拖曳區，允許使用者從本地設備選擇並上傳一張圖片作為編輯對象。
支援的圖片格式應至少包含 .jpg, .jpeg, .png, .bmp, .webp。
上傳後，圖片應顯示在中央的預覽畫布區。預覽區的長寬比應固定為 480:800 (3:5)。
FR-1.2：疊圖圖片上傳

系統應提供另一獨立按鈕，用於上傳作為疊加圖層的圖片。為確保透明效果，此功能應提示或預設上傳 .png 格式。
初始狀態： 疊加的圖片上傳後，其初始位置應為畫布正中央，初始大小應自動縮放至畫布寬度的 50%。
FR-1.3：預設範本與版型

系統應提供一個範本選擇區，內含數個預設版型，例如：「上圖下文」、「左圖右文」、「純文字置中」、「全螢幕圖片」等。
使用者選擇範本後，畫布上會出現對應的圖片佔位符和文字框，使用者只需點擊並替換成自己的內容即可。
3.1.2 內容編輯與操作 (Content Manipulation)

FR-2.1：物件通用操作

使用者應能透過滑鼠或觸控點選畫布上的任一物件（圖片或文字框）使其成為當前選中的作用對象。
FR-2.2：圖片/文字旋轉 (Rotation)

提供「向左旋轉 90°」、「向右旋轉 90°」、「水平翻轉 180°」的按鈕，作用於當前選中的物件。
FR-2.3：圖片/文字縮放 (Scaling)

使用者應能透過滑鼠滾輪、操作介面滑桿或物件邊角的控制點來縮放當前選中的物件。
在觸控裝置上，應支援雙指縮放手勢。
FR-2.4：圖片/文字移動 (Panning)

使用者應能透過滑鼠拖曳來移動畫布中的物件。
在觸控裝置上，應支援單指拖曳手勢。
邊界條件： 當物件被移動或縮放到超出 480x800 的畫布範圍時，超出部分在編輯介面中可見，但在最終輸出時將被自動裁切，僅保留畫布內的部分。
FR-2.5：灰階與抖動處理 (Grayscale & Dithering)

提供一個「轉換為灰階」的開關 (Toggle Switch)，啟用後將畫布上所有彩色內容轉換為灰階。此功能採用預設的最佳化演算法。
提供一個獨立的「抖動效果」開關。此開關僅在「轉換為灰階」啟用時才可操作。啟用後，將對灰階圖像套用抖動演算法（如 Floyd-Steinberg），以模擬更平滑的色階過渡，優化在 E-ink 螢幕上的顯示效果。
FR-2.6：文字添加與編輯

提供「新增文字」按鈕，點擊後在畫布中央生成一個預設文字框。
使用者可輸入自訂文字。
提供以下編輯選項：
字體選擇： 提供 3-5 種經授權的免費商用字體（如 Noto Sans, Source Han Sans 等）。
字體大小調整。
樣式設定： 粗體、斜體。
文字顏色選擇： 提供顏色選擇器。注意：最終輸出時，若啟用灰階模式，所有文字顏色都將轉換為對應的灰階值。
FR-2.7：圖層管理

系統應提供一個簡易的圖層面板，以列表形式顯示畫布上的所有物件（圖片和文字）。
使用者可在面板中：
查看所有圖層。
透過滑鼠拖曳圖層來調整其上下堆疊順序。
點擊圖層旁的眼睛圖示來暫時隱藏或顯示該圖層。
FR-2.8：重製功能 (Reset)

提供「重製目前圖層」按鈕，點擊後僅將當前選中的物件恢復到其剛加入畫布時的初始狀態。
提供「全部重製」按鈕，點擊後會彈出對話框進行二次確認，確認後將清空整個畫布，恢復到初始介面。
3.1.3 圖片輸出 (Image Output)

FR-3.1：輸出規格

最終輸出的圖片尺寸必須為 480 x 800 像素。
輸出的圖片格式必須為 .jpg。
FR-3.2：預覽與下載

提供「預覽」按鈕，點擊後在一個彈出視窗中顯示即將生成的最終圖片。
提供「下載」按鈕，點擊後，系統會將畫布上所有可見圖層合成，並彈出一個對話框讓使用者自訂檔案名稱，確認後觸發瀏覽器下載。
3.2 使用者介面 (UI) 與使用者體驗 (UX) 需求
UI-1：色彩與風格

頁面主背景色調採用 Alice Blue (#F0F8FF)，營造清爽、簡潔的科技感。
UI-2：佈局 (Layout)

頁面內容應水平置中，並設定最大寬度為 800px (或更寬以容納側邊欄)。
整體佈局應清晰劃分：左側為功能操作區（包含圖層、文字編輯等），中央為圖片預覽畫布區。
UI-3：元件設計

卡片式設計 (Card Design): 各功能區塊應設計為帶有輕微圓角 (border-radius) 和陰影 (box-shadow) 的卡片。
互動回饋 (Hover Effects): 所有可互動元件（按鈕、圖層列表等）在滑鼠懸停時應有明確的視覺變化。
過場動畫 (Transitions): 介面元素的狀態變化應加入平滑的過場動畫效果 (transition)，提升操作的流暢感。
UI-4：響應式設計 (Responsive Design)

在螢幕寬度較窄的行動裝置上，佈局應自動調整為適合的單欄垂直排列。
所有控制項的大小應確保在觸控螢幕上易於點擊。
圖片與文字操作應完美支援觸控手勢（拖曳、縮放）。
4. 非功能性需求 (Non-Functional Requirements)
4.1 效能 (Performance)
NF-1.1： 頁面首次載入時間應在 3 秒內完成。
NF-1.2： 所有編輯操作應即時反應，不應有肉眼可見的延遲。
NF-1.3： 圖片輸出生成的時間應在 2 秒內完成。
4.2 易用性 (Usability)
NF-2.1： 介面應直觀，特別是藉由範本功能，讓初次使用者無需說明即可完成一個基本作品。
NF-2.2： 所有按鈕和功能標示應清晰明瞭，建議使用「圖示 + 文字」的組合。
4.3 兼容性 (Compatibility)
NF-3.1： 支援 Chrome, Firefox, Safari, Edge 的最新兩個主要版本。
NF-3.2： 支援 iOS 和 Android 系統上的主流行動瀏覽器。
